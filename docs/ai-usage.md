# AI Tool Usage

AI tools were used as assistants throughout the DuelByte Arena project. The team used AI support to move faster during the hackathon while still reviewing, testing, and customizing the final work.

## Planning and Scoping

AI helped break the original problem statement into small implementation steps:

- Browser game foundation
- Player movement
- Shooting and health
- Powerups
- Map selection
- Hazards and obstacles
- Bots and Solo vs Bot
- Final documentation

This helped keep the project focused on a complete playable demo instead of an oversized feature list.

## UI/UX Brainstorming

AI was used to suggest neon arcade presentation ideas, menu flow, map identity, HUD hierarchy, and demo-friendly polish. The team adjusted these ideas to fit the project scope and readability needs.

## Implementation Support

Codex-style coding assistance was used for implementation guidance and code edits during development. This included support for Phaser scene structure, TypeScript entities, input handling, collision logic, powerup behavior, round flow, and documentation organization.

## Debugging Support

AI was used to reason through likely issues such as scene transitions, reset flow, projectile ownership, obstacle collision, timers, and keeping classic Local Duel stable while adding stretch features.

## Balancing Ideas

AI helped suggest safe starting values for cooldowns, damage, powerup duration, hazard timing, bot behavior, and demo pacing. Final tuning remained practical and team-reviewed.

## Documentation Drafting

AI assisted with drafting final project documentation, including README structure, demo script, game design notes, technical design notes, and testing checklist.

## Team Ownership Statement

AI was used as an assistant, not as a replacement for team ownership. The final game design, feature scope, codebase, testing, demo choices, and submission decisions were reviewed and customized by the team.
